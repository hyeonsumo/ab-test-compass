## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | AB Compass |
| 개발 기간 | 2026.03 ~ 2026.05 |
| 구성 | 개인 프로젝트, 1인 개발 |
| 핵심 기술 | Python, scipy, CustomTkinter, Gemini API |
| 산출물 | Windows 데스크톱 애플리케이션 (`.exe`) |

AB Compass는 A/B 테스트의 그룹 배정, 전환 데이터 시뮬레이션, 통계 분석, 데이터 품질 진단을 직접 구현한 데스크톱 도구입니다.

Statsig, Hackle, Eppo 같은 실무 A/B 테스트 플랫폼이 자동화해 주는 핵심 통계 처리 과정을 직접 코드로 구현해, 결과가 계산되는 원리와 실험 운영 시 주의해야 할 지점을 학습하고 검증하기 위해 만들었습니다.

Gemini API 기반의 AI 실험 설계 도우미도 포함해 자연어 아이디어를 구조화된 A/B 테스트 설계안으로 바꿀 수 있습니다.

## 2. 프로젝트 배경

실무 A/B 테스트 도구는 결과 화면을 빠르게 제공하지만, p-value, 신뢰구간, 표본 수, 데이터 품질 경고가 내부에서 어떻게 계산되는지 자세히 드러나지 않는 경우가 많습니다.

이 프로젝트는 다음 질문을 직접 구현으로 확인하기 위해 시작했습니다.

- 동일 사용자를 항상 같은 그룹에 배정하려면 어떤 방식이 필요한가?
- p-value와 95% 신뢰구간은 어떤 가정과 표준오차에 기반해 계산되는가?
- 50:50으로 설정한 실험에서 53:47처럼 치우친 배정은 언제 문제로 판단할 수 있는가?
- 표본이 충분하지 않은 상태에서 결과를 보고 실험을 종료하면 어떤 오류가 생기는가?
- 자연어 실험 아이디어를 실제 검증 가능한 실험 설계로 바꾸려면 어떤 정보가 필요한가?

## 3. 핵심 구현 내용

### 3-1. 결정적 그룹 배정

무작위 배정처럼 보이지만 동일 사용자는 다시 접속해도 항상 같은 그룹에 들어가야 합니다. AB Compass는 사용자 ID와 실험별 salt를 해시해 상태 저장 없이 일관된 그룹 배정을 수행합니다.

```python
def assign_variant(user_id, salt, ratio):
    key = f"{user_id}:{salt}"
    bucket = int(hashlib.md5(key.encode()).hexdigest(), 16) % 10000 / 10000
    return "A" if bucket < ratio else "B"
```

- **일관성**: 동일 사용자는 언제 접속해도 같은 그룹에 배정됩니다.
- **독립성**: 실험별 salt를 분리해 여러 실험 간 배정 상관을 줄입니다.
- **무상태성**: 별도 서버 조회 없이 사용자 ID만으로 즉시 배정할 수 있습니다.

### 3-2. 통계적 유의성 검정과 신뢰구간

두 그룹의 전환율 차이가 우연인지 실제 효과인지 판단하기 위해 2-표본 z-검정을 구현했습니다. 효과 크기 추정에는 그룹별 분산을 분리해 표준오차를 계산하고, 95% 신뢰구간을 함께 제시합니다.

| 산출 지표 | 설명 |
|---|---|
| p-value | 관측된 차이가 우연히 발생할 확률을 추정합니다. |
| z-score | 두 비율 차이를 표준화한 검정 통계량입니다. |
| 95% 신뢰구간 | 효과 크기의 불확실성 범위를 보여줍니다. |
| Lift | 기준 그룹 대비 실험 그룹의 상대 개선율입니다. |

### 3-3. 데이터 품질 진단

실험 설정 비율과 실제 배정 비율이 일치하는지 카이제곱 검정으로 확인합니다. 이 검정은 Sample Ratio Mismatch(SRM)를 탐지하기 위한 절차입니다.

- SRM 임계값은 `p < 0.001` 기준을 사용합니다.
- 임계값을 넘으면 데이터 수집, 배정 로직, 이벤트 파이프라인 점검이 필요하다는 경고를 표시합니다.
- 실무 도구의 Data Quality Alert와 유사한 진단 흐름을 학습 목적으로 구현했습니다.

### 3-4. 표본 충분성 및 엿보기 경고

권장 표본 수에 도달하기 전에 p-value를 반복적으로 확인하고 실험을 종료하면 false positive가 증가할 수 있습니다. AB Compass는 검정력 0.8 기준의 권장 표본 수를 계산하고, 표본 부족 상태에서는 결과 해석 경고를 표시합니다.

효과가 없는 실험도 중간중간 p-value가 일시적으로 0.05 아래로 내려갈 수 있습니다. 이런 순간을 종료 기준으로 삼으면 명목 유의수준보다 더 높은 오류율이 발생할 수 있으므로, 조기 판단을 막기 위한 안내를 포함했습니다.

### 3-5. 참고 문헌 및 학습 자료

- Kohavi et al., *Trustworthy Online Controlled Experiments* (2020)
- 카카오페이 기술블로그의 A/B 테스트 플랫폼 MVP 접근 사례
- 토스 기술블로그의 실험 기반 의사결정 및 데이터 분석 흐름
- Hackle 기술블로그의 실무 A/B 테스트와 통계 판단 사례

## 4. 주요 기능

### 결정적 그룹 배정

- MD5 기반 해시로 사용자를 A/B 그룹에 일관되게 배정
- 실험별 salt를 사용해 여러 실험 간 배정 독립성 확보

### 트래픽 시뮬레이션

- 기준 전환율과 예상 효과를 입력해 가상 노출 및 전환 이벤트 생성
- seed 고정으로 동일 조건의 실험 재현 가능

### 통계 분석

- 2-표본 z-검정으로 p-value 산출
- 95% 신뢰구간과 lift 계산
- 카이제곱 검정으로 SRM 자동 탐지
- 권장 표본 수 계산 및 표본 부족 경고

### AI 실험 설계

- 자연어 아이디어를 가설, 실험 변형, 측정 지표, 권장 기간, 주의사항으로 구조화
- 통제 변수와 검증 가능성에 대한 사전 점검 프롬프트 제공
- 생성된 설계안을 실험 입력값으로 활용 가능

### 결과 시각화

- 그룹별 전환율 비교 막대 차트
- 누적 전환율 추이 차트
- 통계 분석 결과와 데이터 품질 경고를 함께 표시

## 7. 실행 방법

### 사전 요구사항

- Python 3.10 이상
- 선택 사항: Google Gemini API 키

### 소스 실행

```bash
git clone https://github.com/hyeonsumo/ab-test-compass.git
cd ab-test-compass

python -m venv venv
venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

pip install -r requirements.txt
python app.py
```

### Windows 실행 파일

빌드된 실행 파일이 있는 경우 `dist/ABCompass.exe`를 더블클릭해 실행할 수 있습니다.

### Gemini API 설정

AI 실험 설계 기능은 선택 기능입니다.

1. https://aistudio.google.com/apikey 에서 Gemini API 키를 발급합니다.
2. 앱 실행 후 사이드바의 **AI 설정**에서 키를 입력합니다.
3. 입력된 키는 로컬 `data/config.json`에 저장됩니다.

## 8. 향후 과제

핵심 A/B 테스트 메커니즘 구현 이후 확장해 볼 수 있는 주제입니다.

- **CUPED**: 사전 데이터를 활용해 분산을 줄이고 필요한 표본 수를 낮추는 기법
- **순차 검정**: 반복 확인의 영향을 줄이며 조기 종료를 허용하는 방법
- **베이지안 A/B 분석**: "B가 우세할 확률"처럼 확률 기반으로 결과를 해석하는 방식
- **멀티암드 밴딧**: 실험 중 성과가 좋은 변형에 트래픽을 점진적으로 더 배분하는 방식
- **실험 운영 기능**: SDK, API, 피처 플래그, 권한 관리, 실시간 대시보드
