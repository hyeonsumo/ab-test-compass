## 프로젝트 개요

상용 A/B 테스트 툴을 직접 만들어 보며 핵심 기능을 하나씩 구현하고, 그 과정에서 통계가 실제로 어떻게 동작하는지 이해하기 위해 시작한 프로젝트입니다.

| 항목 | 내용 |
|---|---|
| 개발 기간 | 2026.03 ~ 2026.05 |
| 구성 | 개인 프로젝트, 1인 개발 |
| 기술 스택 | Python, scipy, pandas, CustomTkinter, matplotlib, Gemini API |
| 산출물 | Windows 데스크톱 애플리케이션 (`.exe`) |

**핵심 기능 한눈에 보기**

| 기능 | 설명 |
|---|---|
| 결정적 그룹 배정 | 해시로 사용자를 일관되게 A/B 그룹에 배정 |
| 통계적 유의성 검정 | z-검정과 신뢰구간으로 실험 효과 판단 |
| 데이터 품질 진단 | SRM과 표본 충분성을 확인해 잘못된 결론 방지 |
| AI 실험 설계 | 자연어 아이디어를 실험 설계안으로 자동 변환 |

## 기존 A/B 테스트 툴과의 비교

| 항목 | 상용 플랫폼(Statsig, Hackle, Eppo) | AB Compass |
|---|---|---|
| 그룹 배정 | 결정적 해싱을 내부에서 자동 처리 | MD5 해싱으로 직접 구현 |
| 유의성 검정 | p-value 자동 산출 | z-검정과 신뢰구간 직접 구현 |
| 데이터 품질 | Data Quality Alert 제공 | 카이제곱 검정으로 SRM 직접 진단 |
| 표본 및 검정력 | 권장 표본 안내 | 필요 표본과 엿보기 경고 구현 |
| 고급 통계 및 운영 | CUPED, SDK, 실시간 대시보드 제공 | 이번 범위에서는 제외 |
| 목적 | 팀 단위 SaaS 운영 | 학습용 1인 데스크톱 도구 |

> **통계 엔진 직접 구현**: 랜덤 배정, 유의성 검정, 신뢰구간을 상용 도구와 유사한 원리로 구현했습니다.
>
> **신뢰도 자동 점검**: SRM, 표본 충분성, 조기 종료 위험을 함께 확인해 성급한 결론을 막습니다.
>
> **AI 기반 실험 설계**: 생성형 AI로 자연어 아이디어를 가설, 지표, 기간, 주의사항이 포함된 설계안으로 바꿉니다.

## 전체 워크플로우

```text
실험 설계 -> 그룹 배정 -> 트래픽 시뮬레이션 -> 통계 분석 -> 결과 시각화
```

| 단계 | 설명 |
|---|---|
| 실험 설계 | 가설, 배정 비율, 기준 전환율, 예상 효과를 입력합니다. |
| 그룹 배정 | 사용자를 A/B 그룹으로 일관되게 배정합니다. |
| 트래픽 시뮬레이션 | 가상 사용자의 노출 및 전환 데이터를 생성합니다. |
| 통계 분석 | 유의성, 신뢰구간, SRM, 표본 충분성을 계산합니다. |
| 결과 시각화 | 전환율 비교와 누적 추이를 차트로 확인합니다. |

실험은 직접 입력하거나 AI 실험 설계 기능으로 만들 수 있습니다. AI는 자연어 아이디어를 가설, 지표, 기간, 통제 변수, 주의사항이 포함된 설계안으로 자동 변환합니다.

## 주요 기능

### 1. 결정적 그룹 배정

같은 사용자는 언제 접속해도 항상 같은 그룹에 배정되어야 합니다. AB Compass는 사용자 ID와 실험별 고유값을 조합한 뒤 MD5로 해싱해 0~1 사이 값으로 변환하고, 설정한 배정 비율에 따라 A/B 그룹을 나눕니다.

```python
def assign_variant(user_id, salt, ratio):
    key = f"{user_id}:{salt}"
    bucket = int(hashlib.md5(key.encode()).hexdigest(), 16) % 10000 / 10000
    return "A" if bucket < ratio else "B"
```

- 동일 사용자는 재방문해도 같은 그룹에 배정됩니다.
- 실험마다 salt를 다르게 사용해 여러 실험의 배정이 서로 영향을 주지 않게 합니다.
- 서버에 배정 이력을 저장하지 않아도 동일한 결과를 재현할 수 있습니다.

### 2. 트래픽 시뮬레이션

실제 트래픽 없이도 A/B 테스트에 필요한 노출 및 전환 데이터를 만들 수 있습니다.

- 기준 전환율과 예상 효과를 입력하면 그룹별 전환 확률을 설정합니다.
- 사용자별 노출 및 전환 이벤트를 확률적으로 생성합니다.
- seed를 고정해 같은 조건에서 같은 결과를 재현할 수 있습니다.
- 배포 전에 통계 로직을 빠르게 검증할 수 있습니다.

### 3. 통계 분석

두 그룹의 전환율 차이가 통계적으로 유의한지 판단하고, 결과를 신뢰할 수 있는지 함께 점검합니다.

- 2-표본 z-검정으로 p-value를 계산합니다.
- 95% 신뢰구간으로 효과 크기의 불확실성을 표시합니다.
- lift로 기준 그룹 대비 실험 그룹의 상대 개선율을 보여줍니다.
- 카이제곱 검정으로 배정 비율 이상(SRM)을 진단합니다.
- 권장 표본 수 미달 및 조기 종료 위험을 경고합니다.

```text
예시 결과: p-value 0.0025, z-score 3.02, 신뢰구간 0.6%~3.1%, 유의함, SRM 정상
```

### 4. AI 실험 설계

자연어로 적은 아이디어를 가설, 지표, 실험 기간, 주의사항이 포함된 설계안으로 변환합니다.

```text
입력: "결제 버튼 색상을 바꾸면 전환율이 오를까?"

출력:
- 가설: 버튼 색상 변경이 결제 전환율을 높일 것이다.
- 측정 지표: 결제 전환율(결제 완료 수 / 노출 수)
- 실험 기간: 14일
- 주의사항: 버튼 외 다른 UI 요소는 동일하게 유지
```

AI 설계안은 그대로 실험 입력값으로 활용할 수 있어, 실험 설계 경험이 적어도 빠르게 초안을 만들 수 있습니다.

## 주요 화면

### 실험 설계

실험 이름, 배정 비율, 기준 전환율, 예상 효과, 기간을 입력해 실험을 정의합니다.

- 배정 비율, 기준 전환율, 예상 효과, 기간 설정
- 입력한 비율을 해시 기반 그룹 배정에 적용
- 검증할 가설을 필수 항목으로 기록

### AI 실험 설계

자연어 아이디어를 구조화된 실험 설계안으로 변환합니다.

- Gemini가 아이디어를 정형 실험 설계로 변환
- 통제 변수와 측정 가능성을 사전 점검
- 생성된 설계안을 실험으로 즉시 등록

### 실험 대시보드

실험을 관리하고 가상 트래픽을 생성해 그룹별 지표를 확인합니다.

- 사용자 수 입력 후 노출 및 전환 이벤트 생성
- A/B 그룹의 노출, 전환, 전환율 비교
- 가설, 배정 비율, 기간 등의 핵심 정보 요약

### 결과 분석

수집된 표본으로 유의성과 데이터 품질을 함께 판단합니다.

- p-value, z-score, 신뢰구간 산출
- 카이제곱 검정으로 SRM 진단
- 권장 표본 미달 시 종료 보류 안내

### 결과 시각화

그룹별 전환율 차이와 누적 전환율 변화를 차트로 확인합니다.

- A/B 전환율 막대 차트
- 누적 전환율 추이 차트
- 조기 판단 위험을 추이 기반으로 확인

## 실행 방법

**소스 실행**

```bash
git clone https://github.com/hyeonsumo/ab-test-compass.git
cd ab-test-compass

python -m venv venv
venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

pip install -r requirements.txt
python app.py
```

**Windows 실행 파일**

빌드된 실행 파일이 있는 경우 `dist/ABCompass.exe`를 더블클릭해 실행할 수 있습니다.

**AI 어시스턴트 활성화**

1. [Google AI Studio](https://aistudio.google.com/apikey)에서 Gemini API 키를 발급합니다.
2. 앱 사이드바의 **AI 설정**에서 API 키를 입력합니다.
3. 입력된 키는 로컬 `data/config.json`에 저장됩니다.
